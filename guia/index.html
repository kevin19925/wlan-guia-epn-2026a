En esta primera lección entiendes por qué existe WLAN, qué medios usa, quién estandariza y cómo evolucionó Wi-Fi hasta 802.11bn. Todo lo demás del curso se apoya aquí.

PDF 1 · Medios de transmisión no guiados

Las WLAN se diferencian de Ethernet porque la capa física usa medios no guiados:

  • Infrarrojo — LEDs 850–950 nm, pulsos ~250 ns, hasta ~20 m (DFIR legacy).
  • Luz visible (VLC) — IEEE 802.15.7, 380–780 nm, WPAN multimedia.

PDF 2 · SSID y BSSID

  • SSID — nombre lógico, hasta 32 caracteres, case-sensitive. Comparable a grupo de trabajo Windows.
  • Ocultar SSID — NO está en 802.11-2020 como seguridad; es opción de fabricante, débil.
  • BSSID — en infraestructura = MAC del AP. En IBSS = dirección aleatoria. ≠ SSID.
  • BSA / ESA — área de cobertura de un BSS / de un ESS.

Factores que afectan tamaño BSA: potencia TX, ganancia antena, sensibilidad RX, entorno físico.

PDF 2 · Sistema de distribución (DS)

  • DSM — medio físico del DS (típico: Ethernet 802.3 entre APs).
  • DSS — servicios: asociación, reasociación, integración, distribución MSDUs.
  • El estándar no define el DS — puede ser cualquier LAN.

El DS permite: transferir MSDU entre APs del ESS, entre portal y BSS, y multidestino dentro del BSS.

P1 · EIRP — ejercicio resuelto paso a paso

Enunciado: Pt = 100 mW, pérdidas cable 3 dB, antena 16 dBi. ¿EIRP?

  1. Pt = 100 mW = 20 dBm (10·log₁₀(100) = 20)
  2. Pérdidas 3 dB restan → −3
  3. Ganancia antena 16 dBi suma → +16
  4. EIRP = 20 − 3 + 16 = 33 dBm
  5. mW = 10^(33/10) = 10^3.3 ≈ 1995 mW ≈ 2000 mW

PDF 3 · RSNA — Robust Security Network Association

C1 ¿Qué es un MPDU?

✓ Respuesta: Trama MAC completa = cabecera + body + FCS.

¿Por qué?

El examen mezcla siglas: MPDU es lo que encapsula el PLCP como PSDU. Sin esto no calculas overhead ni η_PLCP.

C2 ¿PSDU y MPDU son lo mismo?

✓ Respuesta: Mismo contenido, distinto nombre: PSDU = MPDU visto desde la capa física.

¿Por qué?

MSDU→MPDU en MAC. PLCP recibe el MPDU y lo llama PSDU. Los bits son idénticos; cambia la capa que los nombra.

¿Por qué?

Cada trama física (RTS, CTS, DATA, ACK) es un PPDU distinto. Por eso pagas T_PLCP cuatro veces en RTS/CTS.

✓ Respuesta: A-MSDU agrega MSDUs con un ACK; A-MPDU agrega MPDUs con Block ACK.

¿Por qué?

Capa física PMD — modulación y MIMO

PDF 4–6 · CCK, OFDM, MCS, MIMO (b, g/n, ac/ax/be).

★ PRUEBA 3
⭐ PDF 4–6 — ÉNFASIS P3 — PMD b/g/n + ac/ax/be: MU-MIMO, OFDMA, RU, tabla slide 166, 9607/46000 Mbps.

PMD es la modulación real en RF. PLCP dice cuánto dura; PMD define cómo se codifican los bits. PDF 6 es el más preguntado en completar datos.

802.11b — DSSS/CCK

  • Chips a 11 Mcps; CCK para 5.5 y 11 Mbps.
  • 1 y 2 Mbps: DBPSK/DQPSK sobre DSSS.

OFDM (a/g/n/ac/ax)

  • 64 subportadoras (legacy) o 256 (ax @ 20 MHz).

PDF 10 · Algoritmo CSMA/CA — DCF

  1. Si medio ocupado → esperar hasta idle + DIFS
  2. Si medio libre tras DIFS → iniciar backoff aleatorio [0, CW] × slot
  3. Contador decrementa solo cuando medio idle
  4. Al llegar a 0 → transmitir
  5. Receptor envía ACK tras SIFS (solo unicast)
  6. Sin ACK → aumentar CW (backoff exponencial) y reintentar

DCF re-contiende en cada nueva transmisión — no mantiene prioridad entre ráfagas.

  2. Rx contesta CTS

L1 ¿Qué es EIRP?

✓ Respuesta: Potencia efectiva radiada isotrópicamente (dBm en antena equivalente).

¿Por qué?

EIRP = Pt(dBm) − pérdidas(dB) + ganancia antena(dBi). Regula potencia legal.

L2 ¿100 mW en dBm?

✓ Respuesta: 20 dBm.

¿Por qué?

10·log₁₀(100)=20. Tabla: 1mW=0 · 10mW=10 · 100mW=20 · 1W=30 dBm.

L3 ¿Pérdida 3 dB?

✓ Respuesta: Potencia se reduce a la mitad (÷2).

¿Por qué?

3 dB≈÷2 · 10 dB≈÷10. En EIRP Prueba3: cable −3 dB después de 100 mW.

L4 ¿Por qué convertir ×N a dB?

✓ Respuesta: En dB todo suma: ganancias (+) y pérdidas (−).

¿Por qué?

13.2× NO es 13.2 dB — es −10·log₁₀(13.2)≈−11.21 dB. Error #1 en cadena.

L5 ¿Chips PN FHSS Prueba3?

✓ Respuesta: 14 chips (AB 100 MHz / espaciado 10 kHz → 2¹⁴).

¿Por qué?

N=10000 canales → menor potencia de 2 ≥ N → k=14.

L6 ¿dBi vs dBm?

✓ Respuesta: dBm=potencia absoluta. dBi=ganancia relativa de antena.

¿Por qué?

EIRP mezcla ambos: dBm tx + dBi antena − dB pérdidas.

❓ Preguntas teóricas — clic para ver respuesta y por qué

S1 ¿Qué significa NAV?

✓ Respuesta: Network Allocation Vector — contador virtual de µs.

¿Por qué?

Implementa VCS. Se carga con Duration de RTS/CTS/Data.

S2 ¿Qué significa EHT?

✓ Respuesta: Extremely High Throughput — PHY de 802.11be (Wi-Fi 7).

¿Por qué?

Sucesor de HE (802.11ax).

S3 ¿Qué significa HE?

✓ Respuesta: High Efficiency — PHY de 802.11ax (Wi-Fi 6).

¿Por qué?

Introduce OFDMA y MU-MIMO bidireccional.

S4 ¿PSDU vs PPDU?

✓ Respuesta: PSDU = MPDU; PPDU = PLCP + PSDU.

¿Por qué?

PPDU es lo transmitido al aire completo.

✅❌ Verdadero / Falso — tipo examen

VF-S1 V/F — MSDU incluye la cabecera MAC.

✓ Falso

¿Por qué?

MSDU = payload antes de MAC. MPDU añade cabecera.

VF-S2 V/F — OFDMA debutó en 802.11ax.

✓ Verdadero

¿Por qué?

ac usa OFDM multiusuario distinto; OFDMA formal en ax.

VF-S3 V/F — SIFS es el IFS más largo del DCF.

✓ Falso

¿Por qué?

SIFS es el más corto (10 µs). DIFS es más largo.

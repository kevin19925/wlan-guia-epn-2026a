A1. Eficiencia MAC y PLCP + tiempo RTS/CTS (Prueba3-3)

Enunciado (PDF Prueba3-3): Estaciones A y B, 802.11g, CSMA/CA con RTS/CTS.

  • A: payload 400 B, CW=3
  • B: payload 200 B, CW=1; tras pérdida recalcula CW=5
  • PLCP: preámbulo 72 bits (corto), cabecera 48 bits
  • MAC: cabecera 30 B, FCS 4 B
  • Tasa datos: 22 Mbps, SIFS=10 µs, slot=20 µs

a) Eficiencia encapsulamiento MAC

OverheadMAC=30+4=34bytesηMAC=payload/(payload+34)EstaciónA:400434=0.9217⇒92.17

b) Eficiencia encapsulamiento PLCP

OverheadPLCP=72+48=120bitsEstaciónA:MPDU=434B=3472bitsηPLCP=3472/(3472+120)=96.66

c) Tiempo para B enviar trama y recibir ACK (CW=5)

Paso 1 — PLCP corto:

Tpreamble=721Mbps=72μsTheader=482Mbps=24μsTPLCP=96μs(sesumaaCADAtramafísica)

Paso 2 — Cuerpo @ 22 Mbps:

T(xB)=x×822μsRTS(20B):96+7.27=103.3μsCTS(14B):96+5.09=101.1μsDATA(234B):96+85.09=181.2μsACK(14B):96+5.09=101.1μs

Paso 3 — Secuencia:

DIFS=SIFS+2×slot=10+40=50μsBackoff=CW×slot=5×20=100μsTtotal=DIFS+Backoff+RTS+SIFS+CTS+SIFS+DATA+SIFS+ACK=50+100+103.3+10+101.1+10+181.2+10+101.1=666.7μs≈0.67ms

A2. NAV RTS/CTS (ejercicios varios #2)

Enunciado: 802.11g, RTS/CTS, tramas MAC 128 B a máxima velocidad.

  • Overhead MAC = 34 B
  • PLCP: preámbulo 144 bits, header 48 bits
  • CW A=5, B=3

Datos asumidos del PDF: 128 B = payload → MPDU = 162 B. Máxima velocidad g = 54 Mbps.

PLCP largo @ 1 Mbps:

TPLCP=(144+48)/1=192μs

Tramas @ 54 Mbps:

TCTS=192+14×854=194.1μsTDATA=192+162×854=216.0μsTACK=194.1μsSIFS=10μs

a) NAV del RTS enviado por A

NAVRTS=SIFS+TCTS+SIFS+TDATA+SIFS+TACK=10+194.1+10+216.0+10+194.1=640.2μs≈640μs

b) NAV del CTS recibido en A

NAVCTS=SIFS+TDATA+SIFS+TACK=10+216.0+10+194.1=430.1μs≈430μs

c) Tiempo total para A (CW=5)

Backoff=5×20=100μsTRTS=192+20×854=195μsTtotal=50+100+195+10+194+10+216+10+194=979μs≈0.98ms

A5. Eficiencia PLCP solo (tipo examen múltiple)

Enunciado típico: Estación A envía 345 bytes de datos a 1 Mbps.

PLCP: cabecera 48 bits, preámbulo 144 bits. Overhead MAC = 34 B.

MPDU=345+34=379bytes=3032bitsOverheadPLCP=144+48=192bitsηPLCP=3032/(3032+192)=30323224=0.9404⇒94.04
Truco: A 1 Mbps, preámbulo y datos van a la misma velocidad, pero la fórmula de eficiencia PLCP sigue siendo bits_MPDU / (bits_MPDU + overhead_PLCP).

SECCIÓN B — Preguntas teóricas (tipo examen)

Respuestas basadas en PDFs Teoria 2026A.

---

B1. ¿Cuáles son las tres funciones principales del preámbulo PLCP?

→ Sincronización, indicar inicio de trama, preparar receptor.

B2. ¿A qué velocidad se transmiten preámbulo y cabecera PLCP en DSSS/802.11g?

→ Siempre a 1 Mbps (PDF 7 y 8). Header corto puede ir a 2 Mbps.

B3. ¿Qué indica el campo Duration en una trama?

→ Microsegundos que el medio estará ocupado → NAV para otras STAs.

B4. ¿Para qué sirve RTS/CTS?

→ Resolver terminal oculto: reservar canal antes de enviar datos largos.

B5. ¿Cuántos bytes tiene el FCS de la trama MAC?

→ 4 bytes (CRC-32).

B6. ¿Cuál es el overhead MAC típico en ejercicios?

→ 34 bytes (30 cabecera + 4 FCS).

B7. ¿Qué diferencia A-MSDU de A-MPDU? (PDF 9)

→ A-MSDU agrega MSDUs con un ACK; A-MPDU agrega MPDUs con Block ACK (obligatorio en ac).

B8. ¿Qué tipo de trama es Authentication?

→ Gestión (tipo 00), subtipo 1011.

B9. ¿Qué tramas NO llevan Sequence Control?

→ Tramas de control (ACK, RTS, CTS).

B10. En 802.11g PPDU, ¿cuántos bits tiene preámbulo + cabecera PLCP?

→ 192 bits (144 + 48) en modo largo.

B11. ¿Qué es el NAV?

→ Network Allocation Vector — contador virtual que indica cuándo el canal estará libre.

B12. ¿Cuál IFS usa DCF para acceder al medio?

→ DIFS (después espera backoff si hubo contención).

B13. ¿Cuánto tiempo entre DATA y ACK?

→ SIFS (10 µs en 802.11b/g).

B14. Campo SIGNAL 802.11g para 54 Mbps equivalente OFDM — ¿qué hex?

→ 0x1E (30) indica DSSS-OFDM; para OFDM puro ver tabla 802.11a Rate.

B15. ¿Qué bits del Frame Control indican tipo de trama?

→ Bits 2-3 (00=Gestión, 01=Control, 10=Datos).
